
# Полный план создания ролика в стиле референса

Ниже описан процесс, который решает две задачи:

1. Позволяет вручную собрать ролик в стиле загруженного видео: плотная речь, мемы, реакции, скриншоты, демонстрации интерфейсов и быстрые смысловые склейки.
2. Превращает этот процесс в повторяемую систему, которую затем можно частично или полностью автоматизировать с помощью AI.

Главный принцип: **сначала нужно разобрать смысл речи, затем подобрать визуальную реакцию на каждый смысловой фрагмент, и только после этого заниматься монтажом и эффектами**.

Сам монтаж технически несложный. Основная сложность находится в редакторской работе: понять фразу, определить её функцию, найти точный визуал и правильно поставить его относительно речи.

---

# Часть 1. Что должно получиться в конце

Результатом работы должен быть не только готовый видеоролик. Желательно получить полный производственный комплект:

1. Финальный ролик.
2. Очищенная голосовая дорожка.
3. Проверенная транскрипция с таймкодами.
4. Таблица смысловых фрагментов.
5. Таблица использованных материалов.
6. Папка с мемами, B-roll, скриншотами и графикой.
7. Монтажный проект.
8. Шаблон проекта для следующих роликов.
9. Набор правил, по которым принимались монтажные решения.
10. При автоматизации — структурированный JSON или таблица, из которой программа может собрать черновой таймлайн.

Таким образом, следующий ролик не начинается с нуля. Для него уже существует структура папок, готовые дорожки, текстовые стили, звуковые эффекты, анимации и библиотека визуальных реакций.

# Часть 2. Зафиксировать задачу до начала работы

## Этап 1. Определить формат ролика

До расшифровки и поиска материалов нужно записать основные параметры проекта.

Пример технического задания:

| Параметр            | Решение                                           |
| ------------------- | ------------------------------------------------- |
| Платформа           | YouTube                                           |
| Формат кадра        | 16:9                                              |
| Основное разрешение | 1920×1080                                         |
| Частота кадров      | 25 или 30 кадров в секунду                        |
| Длительность        | Определяется исходной речью                       |
| Основной формат     | Видеоэссе с закадровым голосом                    |
| Темп                | Высокий                                           |
| Визуальный стиль    | Мемы, реакции, скриншоты, интерфейсы              |
| Фон                 | Чёрный                                            |
| Переходы            | В основном жёсткие склейки                        |
| Текст               | Только ключевые слова и цифры                     |
| Общая текстура      | Шум, пыль или плёночная текстура                  |
| Главная цель        | Удержать внимание и донести тезис                 |
| Дополнительная цель | Перевести зрителя к CTA                           |
| CTA                 | Подписка, продукт, курс, Telegram или другая цель |

Эти параметры должны быть записаны в отдельном файле `PROJECT_BRIEF.md`, `PROJECT_BRIEF.docx` или в первой вкладке монтажной таблицы.

## Почему это необходимо

Без зафиксированного формата монтажёр начинает принимать решения случайно:

* в одном месте использует сложную анимацию;
* в другом — длинный стоковый видеоряд;
* затем вставляет мемы;
* потом добавляет субтитры;
* позже меняет общий стиль.

В результате ролик выглядит как набор разнородных решений.

В референсе стиль достаточно однородный: чёрный холст, центральное расположение материалов, частые склейки, мемные реакции, скриншоты и единая текстура поверх изображения.

## Результат этапа

Одностраничный документ, в котором определены:

* площадка;
* разрешение;
* частота кадров;
* длительность;
* тон;
* визуальные правила;
* уровень мемности;
* структура CTA;
* требования к экспорту.

## Критерий готовности

Любой человек, прочитав документ, должен понимать, как примерно будет выглядеть ролик и какие материалы для него потребуются.

## Правила названий файлов

Нельзя использовать названия:

```text
video1.mp4
final.mp4
final2.mp4
final_REAL.mp4
downloaded_video_9382.mp4
```

Следует использовать понятную схему:

```text
MONEY_REACTION_MAN_SURPRISED_01.mp4
CLAUDE_COMPILER_ARTICLE_SCREENSHOT_01.png
COURSE_UI_TERMINAL_DEMO_01.mp4
CTA_TELEGRAM_QR_01.png
FILM_GRAIN_OVERLAY_01.mov
```

Для версий проекта:

```text
PROJECT_v001
PROJECT_v002
PROJECT_v003
```

Для экспортов:

```text
PROJECT_REVIEW_v003_2026-07-13.mp4
PROJECT_FINAL_MASTER_1080P.mp4
```

## Зачем это нужно

При плотном монтаже может использоваться 100–300 материалов. Без структуры поиск одного клипа занимает больше времени, чем сама его установка на таймлайн.

Кроме того, нормальные названия нужны для автоматизации. AI-система не должна работать с файлами `VID_3847.mp4`; она должна понимать, что это реакция удивления или демонстрация интерфейса.

## Результат этапа

Подготовленная папка проекта, куда каждый материал будет складываться по назначению.

## Критерий готовности

Любой файл можно найти максимум за несколько действий, не просматривая всю папку подряд.

# Часть 4. Подготовить исходную речь

## Этап 3. Выбрать главную аудиодорожку

В ролике такого типа голос является основой. Визуал строится вокруг речи, а не наоборот.

Нужно определить:

* используется закадровый голос;
* используется речь человека в кадре;
* используется комбинация;
* есть ли несколько дублей;
* есть ли ошибки, повторы и длинные паузы;
* нужно ли менять порядок отдельных фраз.

Сначала собирается чистовая версия речи. До этого нельзя точно расставить визуальные материалы, потому что после изменения текста все таймкоды сдвинутся.

## Действия

### 3.1. Импортировать аудио

Поместить исходную запись на отдельную дорожку.

Не добавлять музыку, эффекты и мемы.

### 3.2. Прослушать запись целиком

На первом прослушивании нужно отметить:

* ошибки;
* повторы;
* неудачные дубли;
* длинные паузы;
* кашель;
* щелчки;
* посторонние шумы;
* места, где мысль сформулирована непонятно;
* потенциально сильные фразы;
* места для будущего CTA.

### 3.3. Собрать чистовой дубль

Удаляются:

* повторные попытки сказать одну фразу;
* случайные остановки;
* фразы вроде «эээ», если они не нужны для естественности;
* технические комментарии;
* паузы, возникшие из-за чтения текста;
* посторонние звуки.

Нельзя удалять абсолютно все вдохи и микропаузу между мыслями. Иначе голос станет искусственным и утомительным.

### 3.4. Отрегулировать темп

В референсе речь очень плотная. Однако плотность достигается не ускорением всего аудио, а удалением пустых промежутков.

Практическое правило:

* внутри одного предложения паузы короткие;
* между двумя связанными предложениями пауза немного длиннее;
* перед важным выводом можно оставить микропаузу;
* перед шуткой полезна короткая подготовительная пауза;
* после сильной шутки иногда нужно дать реакции прожить ещё несколько кадров.

### 3.5. Зафиксировать аудиомонтаж

После утверждения текста чистовая речь сохраняется отдельно:

```text
VOICE_LOCKED_v001.wav
```

После этого нельзя случайно продолжать резать исходную дорожку во время визуального монтажа.

## Результат этапа

Чистая, логичная и плотная голосовая дорожка.

## Критерий готовности

Аудио можно включить без изображения, и зритель всё равно понимает:

* о чём ролик;
* какой главный тезис;
* какие приводятся доказательства;
* к какому выводу приходит автор;
* что ему предлагают сделать в конце.

# Часть 5. Очистить и обработать голос

## Этап 4. Техническая обработка речи

Сначала исправляется сам материал. Только после этого применяются компрессор и лимитер.

## Последовательность обработки

### 4.1. Удаление явных дефектов

Исправить:

* щелчки рта;
* удары по микрофону;
* слишком громкие вдохи;
* цифровые клипы;
* короткие фоновые звуки;
* резкие скачки уровня.

### 4.2. Шумоподавление

Шум нужно уменьшать осторожно. Слишком сильное шумоподавление создаёт металлический, подводный или искусственный голос.

Правильный результат: шум перестаёт отвлекать, но голос не теряет естественную фактуру.

### 4.3. Эквализация

Типовая отправная точка:

* фильтр низких частот примерно в районе 70–90 Гц;
* аккуратное уменьшение гула в районе 150–350 Гц;
* небольшое увеличение разборчивости в районе 2–4 кГц;
* контроль резкости в верхней середине;
* при необходимости небольшое добавление воздуха выше 8–10 кГц.

Конкретные значения зависят от голоса и микрофона.

### 4.4. Компрессия

Цель — уменьшить разницу между тихими и громкими словами.

Не нужно превращать речь в полностью плоский сигнал.

Рабочая отправная точка:

* соотношение около 3:1;
* средняя или быстрая атака;
* среднее восстановление;
* уменьшение громкости примерно на 3–6 дБ в громких местах.

### 4.5. Де-эссер

Используется для смягчения чрезмерно резких звуков «с», «ш» и «ц».

### 4.6. Ограничитель

На финальной голосовой шине ограничитель предотвращает цифровые перегрузы.

Пик обычно оставляют ниже 0 dBFS, например около −1 dBTP.

## Результат этапа

Голос звучит одинаково понятно на:

* наушниках;
* ноутбуке;
* телефоне;
* небольшой Bluetooth-колонке.

## Критерий готовности

Не приходится постоянно менять громкость. Согласные понятны, низ не гудит, а шум не отвлекает.

# Часть 6. Сделать полную транскрипцию

## Этап 5. Получить текст с таймкодами

Транскрипция является главным документом всего монтажа.

Нужна не просто расшифровка абзацами, а текст, связанный со временем.

Минимальный формат:

```text
00:00:00.000 — 00:00:03.200
Вайбкодинг убил программирование.

00:00:03.200 — 00:00:06.800
И большинство разработчиков пока не хочет это признавать.
```

Лучший вариант — иметь таймкод каждого слова.

## Последовательность

### 5.1. Сделать автоматическую расшифровку

Система распознавания речи должна вернуть:

* текст;
* начало фразы;
* конец фразы;
* желательно начало и конец каждого слова;
* уверенность распознавания.

### 5.2. Проверить транскрипцию вручную

Особенно внимательно проверяются:

* имена;
* названия компаний;
* названия AI-моделей;
* англоязычные термины;
* числа;
* суммы;
* технические понятия;
* ссылки;
* названия продуктов.

### 5.3. Расставить пунктуацию

Пунктуация должна отражать смысл, а не только паузы в речи.

### 5.4. Сохранить оригинальную формулировку

Не следует незаметно переписывать речь в литературный текст. Для монтажа важны реальные слова, потому что визуал привязывается к конкретному моменту произнесения.

### 5.5. Отдельно создать читаемую версию

Полезно иметь две версии:

1. Дословная транскрипция с таймкодами.
2. Отредактированный текст для понимания структуры.

## Результат этапа

Файл транскрипции, синхронизированный с чистовой аудиодорожкой.

## Критерий готовности

По любому предложению можно быстро определить, на какой секунде оно начинается и заканчивается.

# Часть 7. Разделить речь на смысловые фрагменты

## Этап 6. Создать beat sheet

Обычного деления на предложения недостаточно. Одно предложение может содержать несколько монтажных действий.

Например:

> «Раньше на разработку такого приложения потребовалась бы команда и несколько месяцев, а сейчас один человек может собрать его за выходные».

Внутри находятся минимум четыре смысловых фрагмента:

1. «Раньше».
2. «Потребовалась бы команда».
3. «Несколько месяцев».
4. «Сейчас один человек может собрать его очень быстро».

Каждому фрагменту можно дать отдельный визуал.

## Что такое beat

Beat — минимальная смысловая единица речи, на которую можно поставить отдельное монтажное решение.

Обычно один beat длится от 0,7 до 6 секунд.

Beat может быть:

* утверждением;
* доказательством;
* шуткой;
* вопросом;
* возражением;
* выводом;
* эмоциональной реакцией;
* переходом;
* рекламным сообщением;
* призывом к действию.

## Таблица beat sheet

| Поле           | Содержание                     |
| -------------- | ------------------------------ |
| Beat ID        | Уникальный номер               |
| Time In        | Начало фрагмента               |
| Time Out       | Конец фрагмента                |
| Transcript     | Точный текст                   |
| Short Meaning  | Смысл одной фразой             |
| Function       | Функция в повествовании        |
| Importance     | Важность от 1 до 5             |
| Emotion        | Эмоция                         |
| Keyword        | Ключевое слово                 |
| Visual Type    | Тип визуала                    |
| Search Query   | Поисковая формулировка         |
| Selected Asset | Выбранный материал             |
| Asset In/Out   | Используемый участок материала |
| Motion         | Тип движения                   |
| Text Overlay   | Текст на экране                |
| SFX            | Звуковой эффект                |
| Status         | Статус готовности              |
| Notes          | Комментарий                    |

## Пример строки

| Поле           | Значение                                                         |
| -------------- | ---------------------------------------------------------------- |
| Beat ID        | B023                                                             |
| Time In        | 00:00:47.320                                                     |
| Time Out       | 00:00:50.100                                                     |
| Transcript     | «И разработчики до сих пор делают вид, что ничего не происходит» |
| Short Meaning  | Разработчики игнорируют изменения                                |

| Importance     | 4                                                                |
| Emotion        | Ирония                                                           |
| Keyword        | «ничего не происходит»                                           |
| Visual Type    | Мемная реакция                                                   |
| Search Query   | Человек спокойно сидит во время катастрофы                       |
| Selected Asset | REACTION_DENIAL_03.mp4                                           |
| Motion         | Быстрый punch-in                                                 |
| Text Overlay   | Нет                                                              |
| SFX            | Короткий удар                                                    |
| Status         | Approved                                                         |

## Как делить текст

Новый beat создаётся, когда меняется хотя бы один элемент:

* смысл;
* объект разговора;
* эмоция;
* функция фразы;
* направление аргумента;
* визуальная метафора;
* ключевой персонаж;
* уровень важности.

## Ошибка

Не следует автоматически ставить один клип на каждое предложение.

Иногда одна короткая фраза требует трёх визуальных акцентов. Иногда длинный сложный абзац лучше показать одной понятной демонстрацией.

## Результат этапа

Полная таблица всех смысловых фрагментов ролика.

## Критерий готовности

Для каждого момента речи существует ответ на вопрос:

> Что зритель должен увидеть, почувствовать или понять в этот момент?

# Часть 8. Определить функцию каждой фразы

## Этап 7. Классифицировать beats

Классификация нужна, чтобы не ставить мемы на места, где требуется доказательство.

Основные классы:

| Класс         | Задача                       |
| ------------- | ---------------------------- |
| Hook          | Схватить внимание            |
| Context       | Объяснить ситуацию           |
| Claim         | Высказать тезис              |
| Evidence      | Подтвердить тезис            |
| Example       | Показать конкретный пример   |
| Objection     | Озвучить возражение          |
| Rebuttal      | Ответить на возражение       |
| Consequence   | Показать последствия         |
| Escalation    | Усилить напряжение           |
| Joke          | Разрядить или усилить мысль  |
| Transition    | Перейти к новой теме         |
| Demonstration | Показать продукт или процесс |
| Promotion     | Представить предложение      |
| CTA           | Попросить совершить действие |

## Почему это важно

Фраза «исследование показало рост производительности» требует:

* графика;
* статьи;
* результата эксперимента;
* визуального доказательства.

Мем с удивлённым человеком может усилить эмоцию, но не заменяет доказательство.

Фраза «разработчики в этот момент» может быть показана мемом, потому что её задача — эмоциональная реакция.

## Критерий выбора

Для каждого beat нужно ответить:

1. Зритель должен поверить?
2. Зритель должен понять?
3. Зритель должен почувствовать?
4. Зритель должен рассмеяться?
5. Зритель должен запомнить цифру?
6. Зритель должен увидеть продукт?
7. Зритель должен совершить действие?

От ответа зависит визуальный тип.

# Часть 9. Построить общую драматургию ролика

## Этап 8. Разделить ролик на крупные блоки

До поиска отдельных мемов нужно увидеть всю структуру.

Для ролика такого формата подходит следующая схема.

## Блок 1. Хук

Задача:

* немедленно обозначить конфликт;
* произнести сильный тезис;
* создать ощущение, что происходит важное изменение;
* пообещать объяснение.

В первые секунды желательно дать:

1. Сильное утверждение.
2. Видимую угрозу или возможность.
3. Причину продолжить просмотр.

Визуально:

* наиболее сильные реакции;
* крупные цифры;
* узнаваемые интерфейсы;
* резкие противопоставления;
* быстрые смены кадров.

## Блок 2. Контекст

Задача:

* объяснить, что изменилось;
* представить технологию;
* сформулировать проблему;
* показать старый и новый подход.

Темп остаётся быстрым, но появляется больше понятных демонстраций.

## Блок 3. Доказательства

Задача:

* показать реальные примеры;
* вывести статьи, тесты, графики;
* продемонстрировать интерфейс;
* уменьшить ощущение, что автор просто высказывает мнение.

На этом участке кадры могут быть длиннее, потому что зрителю нужно прочитать и понять информацию.

## Блок 4. Последствия

Задача:

* объяснить, что это означает для зрителя;
* показать, какие навыки становятся менее ценными;
* показать, какие навыки становятся более ценными;
* создать ощущение срочности.

Здесь хорошо работают:

* сравнения;
* мемные реакции;
* денежные образы;
* визуалы профессий;
* контраст «раньше — сейчас».

## Блок 5. Решение

Задача:

* показать, что делать;
* представить метод;
* продемонстрировать продукт или обучение;
* перевести страх или интерес в действие.

Темп обычно немного снижается. Интерфейс продукта должен быть виден достаточно долго.

## Блок 6. CTA

Задача:

* сказать одно конкретное действие;
* визуально показать, куда нажать;
* повторить пользу;
* завершить сильной фразой или шуткой.

CTA не должен содержать пять разных действий одновременно.

Плохой вариант:

> Подпишись, зайди в Telegram, купи курс, оставь комментарий, посмотри другое видео и поставь лайк.

Лучше выбрать одно главное действие и максимум одно дополнительное.

# Часть 10. Построить карту темпа

## Этап 9. Определить монтажную интенсивность

У референса средняя длительность визуального фрагмента составляет примерно 2,36 секунды, медианная — около 1,83 секунды. Около половины фрагментов короче двух секунд.

Это ориентир, а не обязательный норматив.

## Шкала интенсивности

Каждому крупному блоку можно назначить интенсивность от 1 до 5.

| Интенсивность | Описание                          |
| ------------- | --------------------------------- |
| 1             | Спокойная демонстрация            |
| 2             | Объяснение с редкими сменами      |
| 3             | Нормальный плотный монтаж         |
| 4             | Частые смысловые вставки          |
| 5             | Очень быстрый хук или кульминация |

Пример карты:

| Участок                | Интенсивность |
| ---------------------- | ------------- |
| Первые секунды         | 5             |
| Контекст               | 4             |
| Техническое объяснение | 3             |
| Доказательства         | 3–4           |
| Мемная эскалация       | 5             |
| Демонстрация продукта  | 2–3           |
| Финальный CTA          | 4             |

## Примерные длительности визуалов

| Тип материала            | Рабочая длительность |
| ------------------------ | -------------------- |
| Мгновенная реакция       | 0,5–1,5 секунды      |
| Мем с понятным действием | 1–2,5 секунды        |
| B-roll                   | 1,5–4 секунды        |
| Скриншот статьи          | 2,5–6 секунд         |
| График                   | 3–8 секунд           |
| Демонстрация интерфейса  | 3–10 секунд          |
| Текстовая шутка          | 0,8–2 секунды        |
| CTA                      | 3–8 секунд           |

Главный критерий — зритель должен успеть считать необходимую информацию.

# Часть 11. Определить тип визуала для каждого фрагмента

## Этап 10. Использовать дерево решений

Для каждого beat используется следующая логика.

### Фраза содержит проверяемый факт

Использовать:

* статью;
* исследование;
* публикацию;
* официальный интерфейс;
* график;
* документ;
* цифру с источником.

### Фраза описывает конкретный продукт

Использовать:

* логотип;
* сайт;
* интерфейс;
* экранную запись;
* результат работы продукта.

### Фраза описывает действие

Использовать:

* B-roll этого действия;
* экранную запись;
* сцену с аналогичным действием;
* простую визуальную метафору.

### Фраза выражает эмоцию

Использовать:

* реакционный мем;
* лицо;
* сцену из фильма;
* короткий абсурдный клип;
* анимационную реакцию.

### Фраза содержит число

Использовать:

* крупную типографику;
* счётчик;
* график;
* визуальное сравнение;
* объект, позволяющий почувствовать масштаб.

### Фраза содержит сравнение

Использовать:

* разделённый экран;
* «до — после»;
* два последовательных контрастных кадра;
* график;
* параллельные интерфейсы.

### Фраза является шуткой

Использовать:

* реакцию;
* визуальную метафору;
* неожиданный контраст;
* короткую паузу;
* увеличение;
* звуковой акцент.

### Фраза является переходом

Использовать:

* короткий нейтральный B-roll;
* смену главы;
* простую текстовую карточку;
* изменение музыки;
* визуальное перемещение к новому объекту.

# Часть 12. Подобрать ключевое слово для монтажа

## Этап 11. Найти точку синхронизации

Каждый визуал должен быть привязан не просто к началу предложения, а к конкретному слову.

Пример:

> «А потом Claude за один вечер собрал рабочую версию».

Возможные точки:

* на слове «Claude» появляется интерфейс;
* на слове «один вечер» появляется крупный текст;
* на слове «рабочую версию» показывается результат.

## Типы синхронизации

### Визуал появляется точно на слове

Используется для:

* мемного панчлайна;
* названия продукта;
* сильного утверждения;
* числа;
* неожиданных сравнений.

### Визуал появляется немного раньше

Используется, когда зрителю нужно подготовиться к пониманию.

Например, статья появляется за несколько кадров до того, как автор произносит её название.

### Визуал появляется после слова

Используется для реакции.

Автор произносит утверждение, затем на долю секунды появляется реакционный мем как комментарий.

## Ошибка

Плохой монтаж часто запаздывает: автор уже произнёс шутку, а мем появляется через полсекунды. Зритель воспринимает это как несинхронность.

# Часть 14. Создать поисковые запросы

## Этап 13. Искать визуалы не по тексту, а по смыслу

Плохой запрос повторяет слова автора.

Фраза:

> «Разработчики делают вид, что ничего не изменилось».

Плохой запрос:

```text
разработчики делают вид что ничего не изменилось
```

Хорошие запросы:

```text
person ignoring obvious disaster reaction
denial meme calm during chaos
office worker pretending everything is fine
character ignores fire reaction
```

## Формула запроса

```text
[персонаж или объект] + [эмоция] + [действие] + [тип визуала]
```

Примеры:

```text
man shocked looking at screen reaction clip
confused programmer meme
money disappearing animation
person overwhelmed by too many tasks
robot replacing office worker visual metaphor
successful product dashboard screen recording
```

## Для доказательств

Формула:

```text
[точное название] + [тип документа] + [нужный факт]
```

Примеры:

```text
название модели benchmark chart
название компании official announcement
название эксперимента research paper
название продукта interface demo
```

## Для абстрактной мысли

Нужно сначала преобразовать мысль в визуальную метафору.

Фраза:

> «Порог входа резко снизился».

Возможные метафоры:

* открытая дверь;
* сломанный барьер;
* опускающаяся лестница;
* человек легко перепрыгивает препятствие;
* игровая сложность меняется с hard на easy.

# Часть 15. Отбирать материалы системно

## Этап 14. Создать несколько кандидатов на каждый важный beat

Для ключевых фраз не следует сразу брать первый найденный мем.

Нужно сохранить:

* кандидат A — самый буквальный;
* кандидат B — самый эмоциональный;
* кандидат C — самый неожиданный.

Затем выбрать материал с учётом контекста соседних кадров.

## Критерии оценки

Каждому кандидату можно поставить оценки от 1 до 5.

| Критерий            | Вопрос                                                   |
| ------------------- | -------------------------------------------------------- |
| Смысл               | Соответствует ли материал фразе?                         |
| Эмоция              | Передаёт ли нужную реакцию?                              |
| Скорость считывания | Понятен ли он за секунду?                                |
| Новизна             | Не использовался ли похожий кадр недавно?                |
| Качество            | Достаточно ли хорош файл?                                |
| Композиция          | Хорошо ли выглядит на чёрном фоне?                       |
| Монтажность         | Можно ли взять короткий фрагмент?                        |
| Безопасность        | Нет ли лишних логотипов, текста или спорного содержания? |

## Правило разнообразия

Не следует ставить подряд:

* три мема с лицами;
* три скриншота статей;
* три записи терминала;
* три одинаковых зума;
* три вертикальных ролика;
* несколько сцен из одного источника.

Желательно чередовать:

```text
мем → интерфейс → скриншот → реакция → B-roll → текст → демонстрация
```

Это создаёт ощущение постоянного обновления визуальной информации.

# Часть 16. Сделать бумажный монтаж

## Этап 15. Полностью спроектировать ролик до монтажа

Необязательно найти абсолютно все материалы, но для каждого beat должно быть принято предварительное решение.

Пример:

```text
B001 — сильный мем
B002 — крупный текст
B003 — логотип продукта
B004 — экранная запись интерфейса
B005 — реакция недоверия
B006 — статья
B007 — выделение цифры на статье
B008 — мемный вывод
```

## Что фиксируется

Для каждого фрагмента:

* где начинается визуал;
* где заканчивается;
* что он показывает;
* зачем он нужен;
* какой файл используется;
* нужна ли анимация;
* нужен ли текст;
* нужен ли звуковой эффект;
* какой следующий кадр.

## Почему это эффективнее

Без бумажного монтажа работа превращается в постоянные переключения:

1. Прослушать фразу.
2. Остановить монтаж.
3. Открыть браузер.
4. Искать мем.
5. Скачать.
6. Импортировать.
7. Обнаружить, что он не подходит.
8. Начать поиск заново.

При бумажном монтаже поиск и монтаж становятся отдельными этапами.

# Часть 17. Подготовить монтажный проект

## Этап 16. Настроить таймлайн

Рекомендуемая структура видеодорожек:

```text
V6 — дополнительные callout и стрелки
V5 — общая текстура, шум, пыль
V4 — текст и заголовки
V3 — дополнительные реакции и оверлеи
V2 — основной визуал
V1 — фон, подложка, базовые элементы
```

Рекомендуемая структура аудиодорожек:

```text
A1 — основной голос
A2 — дополнительная речь
A3 — музыка
A4 — основные звуковые эффекты
A5 — фоновые и атмосферные эффекты
A6 — room tone или техническая подложка
```

## Настройки последовательности

* разрешение соответствует финальной площадке;
* частота кадров соответствует основному материалу;
* аудио работает в 48 кГц;
* цветовое пространство установлено заранее;
* фон создан отдельным элементом;
* безопасные зоны включены;
* автосохранение настроено;
* прокси создаются при тяжёлых исходниках.

## Создать служебные последовательности

```text
01_VOICE_EDIT
02_ROUGH_CUT
03_FINE_CUT
04_SOUND
05_COLOR_GRAPHICS
06_MASTER
```

Не следует превращать одну последовательность в бесконечный эксперимент.

## Добавить маркеры

Из beat sheet на таймлайн переносятся:

* начало каждого beat;
* ключевые слова;
* начало крупных блоков;
* доказательства;
* мемные панчлайны;
* переход к рекламе;
* CTA.

Маркеры можно раскрасить по функциям:

* красный — хук;
* синий — доказательство;
* зелёный — демонстрация;
* жёлтый — шутка;
* фиолетовый — CTA.

# Часть 18. Собрать первый черновик

## Этап 17. Rough cut

На первом проходе не нужно делать сложные анимации, идеальный звук и цветокоррекцию.

Задача — проверить смысловую последовательность.

## Порядок

### 17.1. Положить чистовую речь

Голос должен быть зафиксирован.

### 17.2. Установить основной визуал для каждого beat

Использовать только одну основную дорожку.

Не добавлять декоративные элементы.

### 17.3. Поставить примерные точки входа и выхода

На этом этапе достаточно точности до нескольких кадров.

### 17.4. Проверить темп

Нужно смотреть:

* нет ли слишком длинных статичных участков;
* успевает ли зритель понять скриншоты;
* не слишком ли часто меняются кадры;
* не конфликтует ли визуал с речью;
* нет ли визуалов, которые отвлекают от важной мысли.

### 17.5. Использовать заглушки

Когда материал ещё не найден, ставится текстовая карточка:

```text
[НУЖНА РЕАКЦИЯ НЕДОВЕРИЯ]
[НУЖЕН СКРИНШОТ СТАТЬИ]
[НУЖНА ДЕМОНСТРАЦИЯ ИНТЕРФЕЙСА]
```

Это лучше, чем останавливать весь монтаж.

## Результат этапа

Полный ролик от начала до конца, даже если некоторые места пока содержат заглушки.

## Критерий готовности

Понятна:

* структура;
* последовательность аргументов;
* общая скорость;
* место каждого доказательства;
* переход к рекламной части;
* финальный CTA.

---

# Часть 19. Сделать точный смысловой монтаж

## Этап 18. Fine cut

На этом этапе каждый кадр синхронизируется с речью.

## Для каждого beat выполняются действия

1. Прослушать фразу несколько раз.
2. Найти ударное слово.
3. Определить точный кадр начала визуала.
4. Определить, должен ли визуал опережать речь.
5. Обрезать пустое вступление исходника.
6. Оставить только момент, который быстро считывается.
7. Проверить связь с предыдущим кадром.
8. Проверить связь со следующим кадром.
9. Отрегулировать длительность.
10. Проверить фрагмент без звука.
11. Проверить фрагмент только по звуку.
12. Проверить в полном контексте.

## Три режима появления визуала

### Прямое подтверждение

Визуал появляется одновременно с названием объекта.

> «Claude Code…»

На слове «Claude Code» появляется интерфейс.

### Предвосхищение

Визуал появляется на несколько кадров раньше.

Это помогает зрителю заранее увидеть контекст.

### Реакция

Визуал появляется после фразы.

Это работает как комментарий монтажёра.

## Микропаузa после панчлайна

Иногда мему нужно дать 4–12 кадров после окончания ключевого слова. Иначе реакция исчезнет раньше, чем зритель её распознает.

## Ошибка

Нельзя подгонять каждый визуал под одинаковую длительность. Это создаёт механический ритм.

---

# Часть 20. Обработать разные форматы материалов

## Этап 19. Разместить горизонтальные, вертикальные и квадратные источники

В референсе материалы разных форматов объединяются чёрным фоном.

## Горизонтальный источник

Варианты:

* заполнить весь кадр;
* оставить чёрные поля;
* слегка уменьшить;
* увеличить важную область;
* использовать окно по центру.

## Вертикальный источник

Не следует всегда растягивать его на весь экран.

Варианты:

* поместить по центру на чёрный фон;
* добавить размытый фон;
* использовать две копии;
* добавить подпись сбоку;
* увеличить центральную часть;
* обрезать, если главный объект остаётся виден.

Для референса более характерен центральный вертикальный материал на чёрной подложке.

## Квадратный источник

Можно:

* центрировать;
* слегка увеличить;
* добавить тень;
* дополнить коротким текстом;
* использовать вокруг свободное пространство как часть дизайна.

## Низкое разрешение

Для слабого исходника:

* не растягивать его чрезмерно;
* уменьшить окно;
* использовать чёрный фон;
* добавить лёгкую текстуру;
* применить умеренное увеличение резкости;
* не пытаться искусственно создать детали, которых нет;
* сделать клип коротким.

---

# Часть 21. Добавить движение

## Этап 20. Анимировать статичные материалы

Главная задача движения — направлять внимание, а не демонстрировать эффект.

## Медленное приближение

Подходит для:

* статей;
* скриншотов;
* фотографий;
* спокойных реакций;
* интерфейсов.

Пример:

```text
Начальный масштаб: 100%
Конечный масштаб: 106–112%
Длительность: 1,5–4 секунды
```

## Быстрый punch-in

Подходит для:

* панчлайна;
* лица;
* неожиданной цифры;
* резкой реакции.

Пример:

```text
Начальный масштаб: 100%
Конечный масштаб: 115–130%
Длительность: 4–10 кадров
```

После punch-in можно:

* удержать кадр;
* немного вернуть масштаб;
* сразу сделать склейку.

## Панорамирование

Используется, когда нужно провести взгляд:

* по статье;
* по длинному интерфейсу;
* от заголовка к цифре;
* от одного объекта к другому.

## Выделение области

Для скриншота:

1. Показать весь экран.
2. Затем приблизить важную область.
3. Добавить рамку или подсветку.
4. Убрать всё лишнее.

## Ограничение

Не следует добавлять движение в каждый кадр. Постоянные зумы перестают ощущаться как акценты.

---

# Часть 22. Добавить текст

## Этап 21. Использовать текст только там, где он усиливает мысль

В референсе текст применяется выборочно.

Основные случаи:

* важная цифра;
* цена;
* название продукта;
* короткий тезис;
* контраст;
* CTA;
* ссылка;
* ключевая шутка.

## Правила

### Один экран — одна мысль

Не следует выводить целый абзац.

Хорошо:

```text
10× БЫСТРЕЕ
```

Плохо:

```text
Благодаря современным инструментам искусственного интеллекта
разработчики могут создавать продукты значительно быстрее
```

### Минимум слов

Обычно достаточно 2–7 слов.

### Высокая читаемость

* крупный размер;
* понятный шрифт;
* сильный контраст;
* достаточное время;
* отсутствие сложного фона;
* безопасные отступы.

### Синхронизация

Текст появляется на ключевом слове или немного раньше.

### Выход текста

В большинстве случаев достаточно:

* появления по opacity;
* быстрого масштаба;
* простой маски;
* жёсткого появления;
* короткого слайда.

Сложная кинетическая типографика для такого стиля необязательна.

## Субтитры

Полные постоянные субтитры можно сделать отдельно, но они меняют характер референса.

Компромисс:

* в основном использовать выборочные акцентные слова;
* полные субтитры подготовить как отдельную дорожку или отдельную версию ролика.

---

# Часть 23. Добавить общую текстуру

## Этап 22. Объединить разнородные материалы

Мемы, статьи, интерфейсы и старые видео имеют разное качество. Общая текстура помогает собрать их в единый визуальный язык.

## Возможные элементы

* мелкое зерно;
* пыль;
* царапины;
* лёгкое мерцание;
* мягкая виньетка;
* небольшая потеря цифровой чистоты.

## Как использовать

Текстура располагается на верхней дорожке поверх всего видео.

Непрозрачность должна быть умеренной. Зритель должен чувствовать фактуру, но не отвлекаться на неё.

## Проверка

Текстура не должна:

* мешать читать статьи;
* портить лица;
* создавать сильное мерцание;
* ухудшать сжатие;
* выглядеть как случайный фильтр.

## Ошибка

Не следует использовать сильное зерно для маскировки плохого исходника. Оно не восстанавливает детали.

---

# Часть 24. Настроить переходы

## Этап 23. Использовать в основном жёсткие склейки

В этом стиле переход создаётся смыслом, а не анимационным пресетом.

Рекомендуемое распределение:

* большинство переходов — обычный cut;
* небольшая часть — быстрый масштаб;
* редкие переходы — flash, blur или whip;
* dissolve — только для осознанной смены состояния;
* затемнение — для главы или финала.

## Когда нужен специальный переход

* резкая смена времени;
* смена крупной главы;
* демонстрация результата;
* переход из обычного повествования в рекламу;
* намеренная комедийная перегрузка.

## Ошибка

Нельзя использовать отдельный эффект между каждыми двумя клипами. Это делает ролик похожим на демонстрацию шаблонов.

---

# Часть 25. Сделать звуковой дизайн

## Этап 24. Добавить звук после стабилизации визуального монтажа

Звуковые эффекты должны поддерживать склейки, но не заменять смысл.

## Категории

### Акцентные удары

Используются для:

* крупной цифры;
* панчлайна;
* появления продукта;
* резкого punch-in;
* важного вывода.

### Whoosh

Используется для:

* быстрого перемещения;
* появления интерфейса;
* короткого слайда;
* перехода между главами.

Не нужен на каждой склейке.

### Click и UI-звуки

Используются для:

* интерфейсов;
* кнопок;
* терминала;
* курсора;
* появления цифровых элементов.

### Комедийные эффекты

Используются осторожно:

* record scratch;
* короткий boing;
* bass drop;
* reverse hit;
* короткий системный звук;
* намеренная тишина.

### Атмосфера

Для длинных демонстраций можно добавить:

* лёгкий компьютерный фон;
* офисную атмосферу;
* слабый шум пространства;
* техническую текстуру.

## Синхронизация

Звук может начинаться:

* точно на склейке;
* на 1–3 кадра раньше;
* в момент начала движения;
* в момент завершения движения.

Иногда звук, начинающийся немного раньше визуала, делает переход естественнее.

## Ограничение

Звуковой эффект не должен перекрывать согласные в речи.

---

# Часть 26. Добавить музыку

## Этап 25. Построить музыкальную динамику

Музыка должна поддерживать структуру, а не конкурировать с голосом.

## Возможная схема

### Хук

* напряжённый или энергичный трек;
* быстрый ритм;
* заметный, но контролируемый уровень.

### Объяснение

* более нейтральная подложка;
* меньше ярких элементов;
* голос выходит на первый план.

### Эскалация

* усиление ритма;
* добавление баса;
* рост плотности.

### Демонстрация продукта

* более чистая и спокойная музыка;
* зрителю дают рассмотреть интерфейс.

### CTA

* возврат энергии;
* музыкальный акцент;
* короткая кульминация.

## Уровень

Музыка обычно должна быть ощутимо тише голоса. Практическая отправная точка — держать её примерно на 12–20 дБ ниже воспринимаемого уровня речи, затем корректировать на слух.

## Ducking

Во время плотных фраз музыку можно опускать на несколько децибел.

Во время пауз и визуальных демонстраций — немного поднимать.

## Ошибка

Нельзя держать один и тот же музыкальный фрагмент восемь минут без изменений. Даже хороший трек начнёт ощущаться как монотонный шум.

---

# Часть 27. Правильно встроить рекламу и CTA

## Этап 26. Сделать переход к продукту логичным

В референсе вторая половина постепенно превращается в рекламную воронку.

Правильная структура:

1. Показать проблему.
2. Показать последствия.
3. Объяснить, что зрителю нужно делать.
4. Представить решение.
5. Показать продукт.
6. Дать доказательство.
7. Сделать CTA.

## Плохой переход

> «AI меняет профессию. Кстати, покупайте мой курс».

## Более естественный переход

> «Поэтому теперь важнее не только знать синтаксис, а уметь управлять агентами. Именно этот процесс мы разбираем внутри продукта».

Затем показываются:

* интерфейс;
* задания;
* терминал;
* результаты;
* сообщество;
* конкретный следующий шаг.

## Визуальная структура CTA

1. Показать продукт.
2. Показать пользу.
3. Показать кнопку, ссылку или QR-код.
4. Крупно повторить действие.
5. Оставить экран достаточно долго.

## Критерий

Зритель должен понимать:

* что ему предлагают;
* для кого это;
* какую проблему это решает;
* что конкретно нужно сделать.

---

# Часть 28. Провести проверку монтажа

## Этап 27. Несколько отдельных просмотров

Нельзя качественно проверить ролик одним просмотром.

## Просмотр 1. Только смысл

Не обращать внимания на эффекты.

Проверить:

* логично ли развивается мысль;
* нет ли повторов;
* понятен ли главный тезис;
* достаточно ли доказательств;
* не появляется ли реклама слишком резко;
* понятен ли финальный вывод.

## Просмотр 2. Только темп

Проверить:

* где становится скучно;
* где слишком много информации;
* где кадры меняются механически;
* где нужен визуальный отдых;
* где важный скриншот исчезает слишком быстро;
* где мем держится слишком долго.

## Просмотр 3. Только синхронизация

Проверить:

* мем появляется на нужном слове;
* удар совпадает с визуальным акцентом;
* текст появляется вовремя;
* реакция не запаздывает;
* музыка не перекрывает голос.

## Просмотр 4. Без звука

Проверить:

* читается ли визуальная логика;
* понятны ли главные объекты;
* нет ли слишком тёмных кадров;
* не прыгает ли композиция;
* читается ли текст;
* достаточно ли разнообразия.

## Просмотр 5. Только звук

Закрыть изображение и проверить:

* понятность речи;
* равномерность громкости;
* уровень музыки;
* отсутствие перегрузов;
* корректность эффектов;
* естественность пауз.

## Просмотр 6. На телефоне

Проверить:

* читаемость мелкого текста;
* размер интерфейсов;
* видимость лиц;
* контраст;
* уровень громкости;
* понятность вертикальных источников.

## Просмотр 7. На повышенной скорости

Просмотр на 1,25× или 1,5× помогает заметить:

* повторения;
* бессмысленные вставки;
* медленные участки;
* перегруженные доказательства;
* лишние паузы.

# Часть 29. Чек-лист каждого кадра

Для каждого визуального фрагмента нужно ответить на семь вопросов:

1. Что именно показывает этот кадр?
2. Как он связан с произносимой фразой?
3. Он объясняет, доказывает или создаёт эмоцию?
4. Зритель успеет его понять?
5. Может ли другой материал выполнить задачу лучше?
6. Не повторяет ли он предыдущие вставки?
7. Ухудшится ли ролик, если этот кадр удалить?

Если после удаления ничего не меняется, кадр, вероятно, лишний.

# Часть 30. Финальная техническая проверка

## Этап 28. Проверить изображение

* нет чёрных кадров;
* нет случайных пробелов;
* нет офлайн-медиа;
* нет водяных знаков;
* нет обрезанных подписей;
* нет текста за безопасной зоной;
* отсутствуют случайные кадры другого проекта;
* нет сильного мерцания;
* вертикальные материалы корректно расположены;
* цвет не прыгает без причины;
* общий grain не мешает чтению.

## Проверить звук

* голос не перегружен;
* музыка не закрывает речь;
* SFX не слишком громкие;
* нет щелчков на монтажных склейках;
* нет внезапных изменений громкости;
* нет пустоты между аудиофрагментами;
* финал не обрывается резко;
* пики остаются в безопасном диапазоне.

## Проверить содержание

* числа написаны правильно;
* названия продуктов написаны правильно;
* интерфейсы соответствуют фразам;
* CTA ведёт в нужное место;
* QR-код читается;
* ссылки актуальны;
* доказательства не искажают смысл источника;
* рекламное обещание сформулировано понятно.

# Часть 31. Экспортировать ролик

## Этап 29. Сделать несколько экспортов

### Review-версия

Для проверки:

* H.264;
* умеренный битрейт;
* быстро загружается;
* содержит номер версии.

### Master-версия

Для публикации:

* 1920×1080;
* частота кадров совпадает с таймлайном;
* H.264 High Profile или другой выбранный мастер-формат;
* VBR;
* ориентировочный битрейт для 1080p — 12–20 Мбит/с;
* AAC;
* 48 кГц;
* высокий аудиобитрейт;
* пик около −1 dBTP.

### Архивная версия

Можно сохранить мастер с меньшей компрессией, чтобы последующие версии не кодировались повторно из уже сжатого файла.

## После экспорта

Экспорт обязательно просматривается целиком.

Нельзя считать файл готовым только потому, что программа завершила рендер.

Проверить:

* начало;
* конец;
* синхронизацию;
* случайные ошибки;
* повреждённые кадры;
* громкость;
* читаемость;
* QR-код;
* отсутствие офлайн-медиа.

---

# Часть 32. Архивировать проект

## Этап 30. Сохранить всё для следующей работы

После публикации нужно:

1. Собрать использованные материалы.
2. Удалить ненужный кэш.
3. Сохранить проект.
4. Сохранить XML, EDL или OTIO.
5. Сохранить финальную транскрипцию.
6. Сохранить beat sheet.
7. Обновить счётчик использования мемов.
8. Добавить удачные материалы в общую библиотеку.
9. Зафиксировать, что сработало.
10. Зафиксировать, что пришлось переделывать.

## Полезный постмортем

Записать:

```text
Что сработало:

— мемы на возражениях;
— демонстрация продукта;
```

# Часть 33. Как превратить процесс в AI-монтажёра

Ручной процесс нужно автоматизировать не сразу целиком, а по слоям.

Правильная последовательность:

```text
Транскрипция
→ смысловое деление
→ классификация фраз
→ предложение визуалов
→ выбор материалов
→ автоматическая черновая сборка
→ шаблонные анимации
→ звук
→ человеческая проверка
```

Система не должна начинать с самостоятельного финального монтажа. Сначала она должна научиться принимать хорошие редакторские решения.

# Часть 41. Модуль поиска и ранжирования

## Модуль 7. Retriever

Система получает beat и возвращает несколько кандидатов.

Пример формулы оценки:

```text
30% — смысловое соответствие
20% — эмоциональное соответствие
15% — скорость считывания
10% — техническое качество
10% — визуальное разнообразие
10% — новизна относительно соседних кадров
5%  — статус источника и использования
```

Пример:

```text
Итоговая оценка =
0.30 × semantic_match
+ 0.20 × emotion_match
+ 0.15 × readability
+ 0.10 × quality
+ 0.10 × diversity
+ 0.10 × novelty
+ 0.05 × source_status
```

## Ограничения

Система должна штрафовать кандидат, если:

* тот же материал уже использован;
* похожая реакция была несколько секунд назад;
* предыдущие два кадра тоже являются мемами;
* материал содержит мелкий текст;
* визуал противоречит факту;
* источник слишком низкого качества;
* важная реакция начинается слишком поздно;
* длительность не подходит.

# Часть 43. Ограничения автоматического монтажа

Система должна соблюдать правила:

1. Не ставить мем вместо доказательства.
2. Не использовать один материал дважды без причины.
3. Не ставить три одинаковых визуальных типа подряд.
4. Не менять кадр в момент, когда зритель читает важный текст.
5. Не держать реакционный мем дольше, чем он остаётся понятным.
6. Не использовать сильную анимацию под сложное объяснение.
7. Не закрывать голос звуковым эффектом.
8. Не ставить текст за безопасной зоной.
9. Не делать резкий CTA без логического перехода.
10. Не использовать высокий темп на протяжении всего видео.
11. Не ставить два разных визуальных акцента на одно короткое слово.
12. Не использовать клип, если главная реакция начинается после его окончания.
13. Не обрезать лицо или главный объект без причины.
14. Не растягивать вертикальное видео до потери композиции.
15. Не показывать мелкий интерфейс, который невозможно прочитать.

# Часть 49. Этапы разработки AI-монтажёра

## Итерация 1. Транскрипция и beat sheet

Система:

1. Загружает видео.
2. Извлекает речь.
3. Создаёт транскрипцию.
4. Делит текст на смысловые фрагменты.
5. Формирует таблицу.
6. Пользователь проверяет.

На этом этапе AI ничего не монтирует.

Цель — добиться качественного понимания структуры речи.

## Итерация 2. Рекомендации визуалов

Система:

1. Классифицирует каждый beat.
2. Предлагает тип визуала.
3. Генерирует запросы.
4. Подбирает материалы из локальной библиотеки.
5. Показывает три кандидата.
6. Пользователь выбирает.

Цель — сократить время поиска.

## Итерация 3. Автоматический rough cut

Система:

1. Берёт утверждённые материалы.
2. Расставляет их на таймлайне.
3. Делает базовые жёсткие склейки.
4. Центрирует кадры.
5. Выдаёт черновой ролик.
6. Экспортирует редактируемый таймлайн.

Цель — получить полный первый монтаж.

## Итерация 4. Анимация и текст

Добавляются:

* slow push;
* punch-in;
* панорамирование;
* выделение области;
* акцентный текст;
* CTA;
* единая текстура.

Цель — приблизить черновик к стилистически готовому видео.

## Итерация 5. Звук

Добавляются:

* очистка речи;
* нормализация;
* музыка;
* ducking;
* звуковые акценты;
* автоматическая проверка громкости.

## Итерация 6. Обучение на правках

Система записывает:

* какие материалы пользователь принял;
* какие отклонил;
* где изменил тайминг;
* какие мемы предпочитает;
* сколько текста оставляет;
* какой темп считает правильным;
* какие эффекты отключает.

После нескольких проектов система начинает учитывать конкретный монтажный вкус.

# Часть 50. Что не нужно автоматизировать первым

Не следует начинать с:

* автоматической генерации сотен переходов;
* сложной кинетической типографики;
* цветокоррекции;
* полностью автоматической публикации;
* самостоятельного поиска любых материалов в интернете;
* генерации финального видео без редактируемого таймлайна.

Самые ценные первые функции:

1. Точная транскрипция.
2. Смысловое деление.
3. Классификация фраз.
4. Поиск подходящих реакций.
5. Синхронизация визуала с ключевым словом.
6. Создание чернового таймлайна.

Именно эти действия занимают основную часть редакторской работы в таком стиле.

# Часть 51. Основные ошибки и способы исправления

## Ошибка 1. Мем на каждую фразу

Проблема: ролик превращается в хаотичную подборку реакций.

Решение: чередовать мемы с доказательствами, интерфейсами, B-roll и визуальными паузами.

## Ошибка 2. Визуал не соответствует смыслу

Проблема: материал подходит по одному слову, но противоречит полной мысли.

Решение: анализировать весь beat и соседние предложения.

## Ошибка 3. Склейка запаздывает

Проблема: реакция появляется после того, как панчлайн уже прошёл.

Решение: привязывать визуал к таймкоду ключевого слова.

## Ошибка 4. Слишком быстрые доказательства

Проблема: статья показывается на одну секунду, и зритель не успевает ничего понять.

Решение: держать доказательство дольше и увеличивать нужный участок.

## Ошибка 5. Одинаковый ритм

Проблема: каждый кадр длится ровно две секунды.

Решение: использовать разные длительности и карту интенсивности.

## Ошибка 6. Слишком много зумов

Проблема: движение перестаёт быть акцентом.

Решение: часть кадров оставлять статичными.

## Ошибка 7. Слишком громкие SFX

Проблема: эффекты перекрывают речь.

Решение: снижать уровень, уменьшать количество и использовать ducking.

## Ошибка 8. Мелкий интерфейс

Проблема: на телефоне невозможно понять, что происходит.

Решение: обрезать интерфейс до нужной области и использовать крупное увеличение.

## Ошибка 9. Резкий рекламный переход

Проблема: зритель чувствует, что повествование внезапно остановилось.

Решение: связать продукт с проблемой, уже разобранной в ролике.

## Ошибка 10. Отсутствие визуального отдыха

Проблема: зритель устаёт от постоянных мемов и ударов.

Решение: после плотного участка использовать более длинную демонстрацию или спокойный кадр.

# Часть 52. Критерии готового ролика

Ролик можно считать готовым, когда выполнены следующие условия:

1. Главный тезис понятен в первые секунды.
2. Речь понятна без изображения.
3. Визуал усиливает речь, а не просто меняется.
4. Каждый важный факт имеет визуальное подтверждение.
5. Каждый мем имеет смысловую или эмоциональную функцию.
6. В начале нет длинных статичных участков.
7. Доказательства остаются на экране достаточно долго.
8. Нет трёх однотипных визуалов подряд.
9. Текст читается на телефоне.
10. Звуковые эффекты не перекрывают голос.
11. Музыка меняется вместе со структурой.
12. Переход к продукту логичен.
13. CTA понятен без дополнительных объяснений.
14. Финальный экспорт проверен целиком.
15. Проект можно открыть и продолжить редактировать.
16. Все материалы организованы и подписаны.
17. Beat sheet соответствует финальному таймлайну.
18. Решения можно повторить в следующем ролике.
